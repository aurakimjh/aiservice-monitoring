## 1. 분석 배경

### 1.1 시장 변화

2024~2026년 AI/LLM 인프라가 기업 핵심 시스템으로 부상하면서, 모니터링·진단 시장에 새로운 수요가 발생했다:

```
전통 IT 모니터링                          AI 시대 모니터링
─────────────────                        ─────────────────
CPU/MEM/Disk                   +         GPU VRAM/SM/Power
HTTP RPS/Latency               +         TTFT/TPS/Token Cost
SQL Query/Slow Query           +         Vector Search/Embedding
Error Rate/5xx                 +         Hallucination Rate/Guardrail Block
App Log                        +         Prompt Log/PII Masking
```

기존 솔루션들은 각자의 강점 영역에서 이 변화에 대응하고 있지만, **IT 인프라 + AI 시스템을 하나의 관점에서 통합 진단**하는 솔루션은 아직 없다.

### 1.2 비교 대상

| 솔루션 | 유형 | 본사 | 주요 고객 |
|--------|------|------|---------|
| **Datadog** | SaaS APM/모니터링 | 미국 | 글로벌 테크 기업 |
| **Dynatrace** | SaaS AI 관측성 | 오스트리아 | 대형 엔터프라이즈 |
| **New Relic** | SaaS 풀스택 관측성 | 미국 | 중소~대형 |
| **Grafana + OTel** | 오픈소스 스택 | 미국/CNCF | DevOps 조직 |
| **WhaTap** | SaaS APM | 한국 | 국내 중소~대형 |
| **Scouter** | 오픈소스 APM | 한국 (LG CNS) | 국내 SI/엔터프라이즈 |
| **AITOP Monitoring** | OTel 기반 AI 모니터링 | 한국 | AI 서비스 운영팀 |
| **AITOP Diagnostic** | AI 자동 진단 | 한국 | IT 컨설팅/진단 팀 |

## 2. 시장 지형도 — 3가지 접근 방식

```
                    실시간 연속 모니터링 (24/7)
                           ▲
                           │
            Datadog ●      │     ● Dynatrace
                           │
         New Relic ●       │       ● WhaTap
                           │
     Grafana+OTel ●        │         ● AITOP Monitoring
                           │
         Scouter ●         │
                           │
                           │
────── 범용 IT ────────────┼──────── AI 특화 ──────
                           │
                           │
                           │              ● AITOP Diagnostic
                           │                (진단 + 보고서)
                           │
                           ▼
                    정기/온디맨드 진단
```

| 접근 방식 | 대표 솔루션 | 특징 | 한계 |
|----------|-----------|------|------|
| **실시간 모니터링** | Datadog, Dynatrace, WhaTap | 24/7 메트릭·트레이스·로그 수집, 대시보드, 알림 | 진단/판정/보고서 없음, AI 특화 메트릭 제한적 |
| **오픈소스 관측성** | Grafana, Prometheus, OTel | 벤더 락인 없음, 커뮤니티 생태계 | 직접 구축 필요, AI 메트릭 표준화 초기 |
| **AI 통합 진단** | AITOP (Diagnostic + Monitoring) | 86개 항목 자동 진단, Rule+LLM 하이브리드, IT↔AI 교차 분석 | 시장 초기, 인지도 구축 중 |

## 3. AITOP 듀얼 플랫폼 전략

AITOP은 성격이 다른 두 플랫폼을 개발하고 있으며, 이 둘이 합쳐질 때 시너지가 발생한다:

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                   │
│  AITOP Diagnostic                    AITOP Monitoring             │
│  (aitop-backend/frontend)            (aiservice-monitoring)       │
│                                                                   │
│  ● 정기/온디맨드 진단                 ● 24/7 실시간 모니터링        │
│  ● 86개 항목 (IT 55 + AI 31)        ● OTel 기반 Traces/Metrics   │
│  ● Rule + LLM 하이브리드 판정        ● 44개 대시보드 페이지        │
│  ● 종합 보고서 자동 생성 (PDF)        ● 12종 Agent Collector       │
│  ● 에이전트 수집 + 수동 업로드        ● GPU/LLM/RAG/VectorDB      │
│  ● 교차 분석 (타임아웃 체인 등)       ● 이상 탐지 + 알림           │
│  ● Agentic AI 대화형 분석             ● AI Copilot (NL→PromQL)    │
│                                                                   │
│         └──────────── 통합 시너지 ──────────────┘                 │
│                                                                   │
│  Monitoring의 OTel 메트릭 → Diagnostic의 Evidence 소스로 활용     │
│  Diagnostic의 진단 결과 → Monitoring 대시보드에 Insight 표시      │
│  Monitoring의 SLO 데이터 → Diagnostic의 Rule Engine 임계치        │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 컨설턴트 관점

> "대부분의 모니터링 솔루션은 **'무엇이 일어나고 있는가'**를 보여주지만, **'왜 일어났고, 어떻게 고쳐야 하는가'**는 알려주지 않는다. AITOP은 이 간극을 메우는 유일한 솔루션이다."

### 고객 관점

> "Datadog으로 GPU 사용률 80%를 보고 있지만, 그게 정상인지 위험인지 모른다. AITOP Diagnostic은 '현재 배칭 설정 대비 GPU 사용률이 비효율적이며, continuous batching 적용 시 30% 개선 가능'이라고 판정해준다."

## 5. 심층 비교 — 솔루션별 분석

### 5.1 Datadog

**강점 (배울 점)**:
- 800+ 통합 — 어떤 기술 스택이든 5분 내 연결 가능
- LLM Observability 기능 최근 출시 (프롬프트 트레이싱, 토큰 비용 추적)
- 통합 플랫폼 UX — 메트릭/트레이스/로그가 하나의 화면에서 연결
- 강력한 알림 체계 (Anomaly Detection, Composite Monitor)

**약점 (AITOP 기회)**:
- 비용이 빠르게 증가 (로그/커스텀 메트릭 추가 과금)
- AI/GPU 메트릭이 아직 초기 단계 — DCGM 수준의 심층 GPU 모니터링 부재
- **진단/판정/보고서가 없음** — "CPU 80%"는 보여주지만 "왜 높고 어떻게 해결하는가"는 없음
- 온프레미스 배포 불가 — 금융/공공 등 SaaS 불가 고객 대응 한계

> **컨설턴트 평가**: "Datadog은 관측성의 표준이다. 하지만 'so what?' — 데이터를 보여주고 끝이다. AITOP Diagnostic의 Rule+LLM 자동 판정은 Datadog 사용자가 수작업으로 하는 분석을 자동화한다."

### 5.2 Dynatrace

**강점 (배울 점)**:
- **Davis AI** — 자동 근본 원인 분석 (AIOps) 업계 최고 수준
- 코드 레벨 자동 계측 (OneAgent) — 설정 없이 전체 스택 가시성
- 애플리케이션 보안 통합 (RASP)
- Smartscape — 서비스 의존관계 자동 탐색 + 시각화

**약점 (AITOP 기회)**:
- 매우 높은 비용 (Host Unit 가격)
- AI/LLM 특화 메트릭 아직 미흡
- **Davis AI는 "원인 추정"이지 "진단 판정 + 개선 권고"가 아님**
- 한국 시장 지원 약함 (한국어 UI/문서 미흡)

> **컨설턴트 평가**: "Dynatrace의 Davis AI는 AITOP이 가장 벤치마킹해야 할 기능이다. 하지만 Davis는 실시간 이상에 대한 원인 분석이고, AITOP은 정기 진단+보고서+개선 권고라는 다른 가치를 제공한다."

### 5.3 New Relic

**강점 (배울 점)**:
- 사용자 기반 과금 — 데이터 인제스트 100GB 무료
- AI Monitoring 기능 (LLM 응답 추적, 비용 분석)
- NRQL — 강력한 쿼리 언어로 모든 데이터를 탐색
- 오픈 소스 에이전트 (Go, Java, .NET, Python, Node.js)

**약점 (AITOP 기회)**:
- GPU 모니터링 기능 약함
- 온프레미스 배포 불가
- AI 모니터링이 LLM API 호출 추적에 한정 (RAG/VectorDB/Agent 미지원)
- **보고서/진단 자동화 없음**

### 5.4 Grafana + OTel (오픈소스 스택)

**강점 (배울 점)**:
- **완전한 벤더 중립** — OTel 표준 기반, 어떤 백엔드로든 교체 가능
- 커뮤니티 생태계 — 수천 개 대시보드, 플러그인
- Grafana 11의 AI/ML 기능 (SiftOps, ML 기반 이상 탐지)
- 비용 효율적 — 자체 인프라에 무료 운영 가능

**약점 (AITOP 기회)**:
- **직접 구축 부담** — Prometheus + Tempo + Loki + Grafana 조합 관리
- AI/LLM 특화 대시보드는 직접 만들어야 함
- 진단/판정/보고서 기능 없음 — 순수 시각화 도구
- 엔터프라이즈 지원 부재 (Grafana Cloud 별도)

> **컨설턴트 평가**: "AITOP Monitoring은 이 오픈소스 스택을 기반으로 AI 특화 레이어를 추가한 것이다. Grafana의 대시보드 생태계를 직접 경쟁하기보다, AI 서비스 운영이라는 틈새에서 차별화하는 전략이 현명하다."

### 5.5 WhaTap

**강점 (배울 점)**:
- **한국 시장 최적화** — 한국어 UI, 국내 기술 지원, 공공/금융 레퍼런스
- XLog(트랜잭션 산점도) — 한눈에 성능 분포를 파악하는 독자적 UX
- 합리적 가격 — 글로벌 SaaS 대비 50~70% 저렴
- 빠른 설치 — 5분 내 에이전트 설치 + 데이터 수집

**약점 (AITOP 기회)**:
- AI/LLM/GPU 모니터링 기능 없음
- 오픈소스가 아님 — 벤더 종속
- 글로벌 시장 인지도 낮음
- **진단/보고서 기능 없음** — 순수 모니터링

> **고객 평가**: "WhaTap은 국내 APM의 강자다. 하지만 AI 인프라 모니터링은 지원하지 않는다. AITOP Monitoring이 WhaTap의 한국 시장 경험(XLog, 한국어, 가격)을 배우면서 AI 특화로 차별화하면 좋겠다."

### 5.6 Scouter

**강점 (배울 점)**:
- **완전 무료 오픈소스** — 국내 SI 프로젝트에서 사실상 표준
- XLog + HeatMap — 트랜잭션 분석의 원조 (WhaTap의 원류)
- Java 에이전트 바이트코드 인젝션 — 코드 수정 없이 계측
- 경량 — 리소스 사용량 최소화

**약점 (AITOP 기회)**:
- Java 중심 — Python, Go, Node.js 지원 미흡
- 분산 트레이싱 미지원 (단일 JVM 중심)
- AI/GPU/LLM 전혀 미지원
- 개발 활동 둔화 (최근 커밋 감소)
- **모니터링만 제공, 진단/보고서 없음**

## 8. AITOP의 차별적 강점

### 8.1 경쟁사 대비 유일한 기능

```
┌─────────────────────────────────────────────────────────────────┐
│                    AITOP만의 차별적 가치                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  1. Rule + LLM 하이브리드 진단                                     │
│     ─ 결정론적 규칙(임계값) + AI 해석(맥락)을 결합                  │
│     ─ 경쟁사: 없음 (Dynatrace Davis는 실시간 이상 탐지, 진단 아님)  │
│                                                                   │
│  2. IT + AI 통합 86개 항목 자동 진단                                │
│     ─ OS/WEB/WAS/DB 55개 + AI/LLM/GPU/RAG 31개                    │
│     ─ 경쟁사: 모니터링만 제공, 진단/판정/권고 없음                   │
│                                                                   │
│  3. IT↔AI 교차 분석                                                │
│     ─ GPU OOM + JVM OOM 동시 탐지, LLM 타임아웃 → HTTP 체인         │
│     ─ 경쟁사: IT와 AI를 별도 도구로 관리, 교차 분석 불가             │
│                                                                   │
│  4. 종합 보고서 자동 생성                                           │
│     ─ 86개 항목 판정 → 도메인별 점수 → PDF/Word 자동 출력           │
│     ─ 경쟁사: 대시보드 스크린샷 수동 캡처가 전부                    │
│                                                                   │
│  5. Agentic AI 대화형 심층 분석                                     │
│     ─ 12개 전문 에이전트가 진단 결과를 기반으로 심층 질의 응답       │
│     ─ 경쟁사: 단순 챗봇 수준 (Datadog Bits AI, Dynatrace Davis)    │
│                                                                   │
│  6. 듀얼 플랫폼 시너지                                              │
│     ─ Monitoring(24/7) + Diagnostic(정기 진단)이 하나의 생태계       │
│     ─ 경쟁사: 모니터링 OR 진단, 둘 다 제공하는 솔루션 없음          │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 10. 결론 — AITOP의 포지셔닝

### "Observe, Diagnose, Improve" — 관측을 넘어 진단과 개선까지

```
경쟁사                            AITOP
─────                            ─────
Observe (관측)                    Observe (관측)      ← Monitoring
  ↓                                ↓
Alert (알림)                      Diagnose (진단)     ← Diagnostic (Rule+LLM 86항목)
  ↓                                ↓
Manual Analysis (수작업)          Recommend (권고)     ← 종합 보고서 자동 생성
  ↓                                ↓
(컨설턴트 투입)                   Improve (개선)       ← Agentic AI 대화형 분석
```

### 에이전트 일원화 결정 (ADR-011)

경쟁 분석의 핵심 교훈: **모든 경쟁사는 에이전트 1개**다. AITOP은 현재 Diagnostic(Java)과 Monitoring(Go) 에이전트가 분리되어 있으나, **Go Agent 기반 단일 에이전트로 일원화**를 결정했다. 상세: [ADR-001_AGENT_UNIFICATION.md](./ADR-001_AGENT_UNIFICATION.md)

AITOP은 모니터링 도구가 아니다. **AI 시대의 IT 운영 자동화 플랫폼**이다.

- **Datadog이 보여주는 것**: "GPU 사용률 80%"
- **Dynatrace가 알려주는 것**: "GPU 사용률이 이상치입니다"
- **AITOP이 하는 것**: "GPU 사용률 80%는 현재 배칭 설정(max_batch=32) 대비 비효율적입니다. continuous batching 적용 시 처리량 30% 향상, VRAM 15% 절감이 가능합니다. 우선순위: High. 관련 항목: ITEM0219, ITEM0220. [보고서 다운로드]"
