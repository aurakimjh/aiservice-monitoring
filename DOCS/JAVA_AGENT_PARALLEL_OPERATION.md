## 1. 병행 운영 개요

Go Agent(aitop-agent v2.0.0)가 Java Agent(aitop-onsite)의 모든 기능을 대체함에 따라,
**3개월간 두 에이전트를 병행 지원**하여 기존 고객의 안정적인 전환을 보장합니다.

| 구분 | 기간 | 정책 |
|------|------|------|
| 병행 운영 | 2026-03-26 ~ 2026-06-25 | Java Agent + Go Agent 동시 지원 |
| 전환 완료 | 2026-06-26 이후 | Go Agent 단독 운영 권장 |
| Java Agent EOL | 2026-09-30 | Java Agent 유지보수 전용 전환 |

## 4. 동시 실행 가이드라인

병행 운영 기간 중 **검증 목적**으로 동일 서버에 두 에이전트를 잠깐 동시 실행할 수 있습니다.

```
⚠️  동시 상시 실행 금지
  두 에이전트를 모두 24/7 상시 실행하면
  동일 데이터가 Collection Server에 이중으로 전송되어
  대시보드 메트릭이 2배로 집계될 수 있습니다.
```

### 허용되는 병행 실행 시나리오

| 시나리오 | 허용 여부 | 권장 기간 |
|----------|:--------:|:--------:|
| Go Agent 기동 검증 (Java Agent 중지 상태) | ✅ | -        |
| 단기 비교 수집 테스트 (< 30분) | ✅ | 전환 초기  |
| 24/7 상시 동시 실행 | ❌ | -        |

## 5. 전환율 모니터링 지표

AITOP 운영팀은 다음 지표를 주간으로 추적합니다.

| 지표 | 목표 (병행 운영 종료 시점) |
|------|--------------------------|
| 전체 고객 중 Go Agent 전환율 | ≥ 90% |
| 신규 고객 Go Agent 채택율 | 100% |
| Java Agent 잔류 고객 수 | ≤ 전체의 10% |
| 마이그레이션 지원 티켓 해결율 | ≥ 95% |

## 6. 에스컬레이션 기준

| 상황 | 대응 |
|------|------|
| Go Agent에서 Java Agent 동등 기능 누락 발견 | 즉시 P0 버그 등록 → 1주 내 패치 |
| 전환 후 수집 데이터 누락 | 전담 기술지원 + 필요 시 Java Agent 임시 복귀 허용 |
| 특수 환경(AIX 등) 미지원 항목 | 개별 스크립트 래핑 구현 지원 |
