## 1. 설계 배경 및 목표

### 1.1 현재 문제점

| 문제 | 상세 | 영향 |
|------|------|------|
| UI 분산 | Grafana 5개 대시보드 + XLog 대시보드 + Jaeger + Prometheus가 별도 URL | 운영자가 4~5개 탭 전환 필수, 컨텍스트 유실 |
| 프로젝트 개념 부재 | 서비스별 대시보드가 있으나 프로젝트(고객/환경) 단위 관리 불가 | 멀티테넌트, 다중 고객 환경 대응 불가 |
| 자원 관리 부재 | 서버, 미들웨어, AI 시스템의 인벤토리 관리 UI 없음 | 어떤 서버에서 어떤 서비스가 돌고 있는지 한눈에 파악 불가 |
| 에이전트 연계 부재 | AITOP 에이전트에서 수집한 진단 데이터와 실시간 모니터링 연계 불가 | 진단 보고서와 모니터링 데이터 분리 |
| 상용 수준 미달 | 검색, 필터링, 커스텀 대시보드, 권한 관리 등 상용 기능 부재 | 솔루션 판매 불가 |

### 1.2 설계 목표

```
핵심 원칙: "한 화면에서 모든 것을 — One Pane of Glass"

1. 통합 중앙화    → 모든 모니터링 데이터를 단일 UI로 집중
2. 프로젝트 중심  → 고객/환경별 프로젝트로 리소스를 조직화
3. 계층적 드릴다운 → 프로젝트 → 호스트 그룹 → 서버 → 서비스 → 트레이스 → 스팬
4. 에이전트 통합  → AITOP Agent 수집 데이터 + OTel 실시간 데이터 통합 표시
5. 상용 품질     → 판매 가능한 수준의 UX, 성능, 확장성
6. AI 네이티브   → AI/LLM 서비스 모니터링을 1등 시민으로 설계
```

### 1.3 대상 사용자 (Persona)

| Persona | 역할 | 주요 관심사 | 사용 빈도 |
|---------|------|-----------|---------|
| **SRE / DevOps** | 인프라 운영, 장애 대응 | 실시간 알림, 서버 상태, 에러율 | 상시 (24/7) |
| **AI Engineer** | LLM/RAG 서비스 개발·운영 | TTFT, TPS, GPU 사용률, 모델 성능 | 매일 |
| **아키텍트 / 컨설턴트** | 시스템 점검, 진단 보고서 | IT-AI 교차분석, 종합 대시보드, 보고서 | 주기적 (주/월) |
| **경영진 / PM** | 서비스 상태 개요, SLO 준수 | Executive 대시보드, 비용, 트렌드 | 주간/월간 |
| **고객 (멀티테넌트)** | 자기 프로젝트 모니터링 | 내 서비스 상태, 알림, 보고서 | 수시 |

## 2. 경쟁 APM 벤치마크 분석

### 2.1 주요 APM 솔루션 UI 특성 비교

| 기능 | Datadog | New Relic | Dynatrace | Grafana | **우리 설계** |
|------|---------|-----------|-----------|---------|------------|
| 통합 네비게이션 | 좌측 사이드바 + 상단 검색 | 좌측 사이드바 + 글로벌 서치 | 좌측 사이드바 + Smartscape | 좌측 사이드바 + 패널 기반 | **좌측 2단 사이드바 + Command Palette** |
| 토폴로지 맵 | Service Map | Distributed Tracing Map | Smartscape 3D | Node Graph | **서비스 토폴로지 맵 (2D/계층)** |
| 프로젝트/조직 | Organization > Team | Account > Sub-account | Environment > Host Group | Organization > Folder | **Organization > Project > Host Group** |
| AI/LLM 지원 | LLM Observability (2024~) | AI Monitoring | Davis AI (AIOps) | ML 플러그인 | **AI 네이티브 1등 시민** |
| 커스텀 대시보드 | 위젯 드래그&드롭 | 위젯 편집기 | 타일 기반 | 패널 + 변수 | **위젯 드래그&드롭 + 템플릿** |
| 검색 | 통합 검색 + 태그 필터 | NRQL 쿼리 | DQL 쿼리 | Explore (PromQL) | **통합 검색 + 쿼리 빌더 + Command Palette** |
| 알림 | Monitor + Incident | Alert Condition + Workflow | Problem + Davis | Alert Rule + Contact Point | **Alert Policy + Incident + Runbook** |
| 에이전트 관리 | Fleet Automation | Guided Install | OneAgent 자동 | Alloy (Agent) | **AITOP Agent Fleet Console** |

### 2.2 벤치마크에서 가져올 Best Practice

```
[Datadog에서 배울 점]
├── 통합 좌측 네비게이션 — 모든 기능을 한 곳에서 접근
├── Service Map — 서비스 간 의존관계와 상태를 한눈에
├── Tag-based 필터링 — 모든 곳에서 일관된 태그 필터
└── Notebook — 조사 과정을 기록하고 공유

[New Relic에서 배울 점]
├── NRQL 쿼리 — 강력한 데이터 탐색 쿼리 언어
├── Workload — 관련 서비스를 논리적으로 그룹화
├── AI Monitoring — LLM 호출별 토큰/비용/성능 추적
└── SLI/SLO 관리 — 서비스 수준 목표 대시보드

[Dynatrace에서 배울 점]
├── Smartscape — 인프라↔서비스↔프로세스 자동 토폴로지
├── Davis AI — AI 기반 근본 원인 분석 (RCA)
├── OneAgent — 단일 에이전트로 전체 스택 자동 계측
└── Problem 카드 — 관련 이벤트를 하나의 문제로 자동 그룹화

[Grafana에서 배울 점]
├── 패널 + 변수 시스템 — 유연한 대시보드 커스터마이징
├── Explore — 데이터 소스별 자유 탐색
├── Alert Manager — 계층적 알림 라우팅
└── 플러그인 생태계 — 확장 가능한 시각화
```

## 부록 A: AITOP 에이전트 연동 데이터 모델

```typescript
// 프로젝트에 등록된 리소스 모델
interface Project {
  id: string;
  name: string;
  description: string;
  environment: 'production' | 'staging' | 'development';
  tags: Record<string, string>;
  hostGroups: HostGroup[];
  services: Service[];
  aiServices: AIService[];
  agents: AgentInfo[];
  alertPolicies: AlertPolicy[];
  sloTargets: SLOTarget[];
}

interface HostGroup {
  id: string;
  name: string;
  hosts: Host[];
}

interface Host {
  id: string;
  hostname: string;
  os: string;
  cpu: number;        // 코어 수
  memory: number;     // GB
  status: 'healthy' | 'warning' | 'critical' | 'offline';
  agent?: AgentInfo;
  middlewares: Middleware[];
  gpus?: GPUInfo[];
}

interface Middleware {
  type: 'web' | 'was' | 'db' | 'cache' | 'mq';
  name: string;       // nginx, tomcat, postgresql, redis, kafka
  version: string;
  port: number;
  status: 'running' | 'stopped' | 'error';
  metrics: Record<string, number>;
}

interface AIService {
  id: string;
  name: string;
  type: 'llm' | 'rag' | 'agent' | 'embedding';
  model?: string;
  hostIds: string[];
  metrics: {
    ttft_p95?: number;
    tps_p50?: number;
    gpu_vram_percent?: number;
    error_rate?: number;
    cost_per_hour?: number;
    guardrail_block_rate?: number;
  };
}

interface AgentInfo {
  id: string;
  hostId: string;
  version: string;
  status: 'healthy' | 'degraded' | 'offline' | 'upgrading';
  plugins: PluginInfo[];
  lastHeartbeat: string;  // ISO 8601
  lastCollection: string;
  mode: 'full' | 'collect-only' | 'collect-export';
}

interface PluginInfo {
  id: string;         // ai-llm-agent, ai-gpu-serving, ai-vectordb-embedding
  version: string;
  status: 'active' | 'inactive' | 'error';
  itemsCovered: string[];  // ITEM0200~ITEM0230
}

// AITOP 진단 결과와의 연동
interface DiagnosticResult {
  id: string;
  projectId: string;
  hostId: string;
  executedAt: string;
  items: DiagnosticItem[];
  crossAnalysis: CrossAnalysisResult[];
  reportUrl?: string;  // PDF/Word 다운로드 URL
}

interface DiagnosticItem {
  itemId: string;      // ITEM0200
  itemName: string;    // 할루시네이션 제어 메커니즘
  category: 'it' | 'ai';
  area: 'stability' | 'efficiency' | 'maintainability';
  severity: 'good' | 'warning' | 'critical';
  score: number;       // 0~100
  summary: string;
  recommendations: string[];
  relatedMetrics: string[];  // OTel 메트릭명 참조
  relatedTraceIds?: string[];
}
```

## 부록 B: 화면 흐름 (User Flow)

```
── SRE 장애 대응 흐름 ──

── AI Engineer 성능 튜닝 흐름 ──
AI 서비스 개요 → LLM 성능 대시보드(TTFT 분포 확인) →
GPU 클러스터 뷰(VRAM 확인) → RAG 파이프라인(검색 품질) →
가드레일 분석 → AITOP 진단 보고서(교차 분석) →
설정 변경 → 효과 모니터링

── 아키텍트/컨설턴트 점검 흐름 ──

수집 대기 → 진단 보고서 확인(86개 항목) →
IT-AI 교차 분석 → PDF 보고서 다운로드 → 고객 전달

── 경영진 주간 리뷰 흐름 ──
홈 → Executive 대시보드 → SLO 준수율 → 비용 추이 →
```
