## 1. 릴리스 게이트 개요

v1.0 GA는 아래 **4축 기준을 모두** 통과해야 한다.

| 축 | 게이트 | 필수/선택 | 판단 주체 |
|----|--------|:--------:|---------|
| **기능 완성도** | 필수 워크스트림 완료 확인 | ★ 필수 | 개발 리드 |
| **성능** | 성능 벤치마크 실측 통과 | ★ 필수 | QA / 벤치마크 CI |
| **안정성** | 안정성 시나리오 전체 통과 | ★ 필수 | QA / 운영팀 |
| **문서/배포** | 상용 배포 패키지 완성 | ★ 필수 | 릴리스 매니저 |

> **판정 규칙**
> - ★ 필수 항목 전부 PASS → v1.0 GA 승인 가능
> - ★ 필수 항목 1건이라도 FAIL → 패치 후 재검증 필요
> - ☆ 권고 항목은 80% 이상 PASS 권장 (미달 시 Known Issues 등재)

## 4. 안정성 기준 (R-12)

### 4.1 장애 복구 (Fault Recovery)

| 시나리오 | 합격 기준 | 측정 방법 | 우선순위 |
|---------|---------|---------|:-------:|
| Collection Server 재시작 | 재시작 후 30초 내 수집 재개 | E2E 자동 테스트 | ★ |
| Agent → Server 연결 끊김 | 자동 재연결, 데이터 손실 < 1분 분량 | 네트워크 차단 시나리오 | ★ |
| SQLite 파일 잠금 충돌 | 에러 없이 큐 대기 → 재시도 | 동시 쓰기 테스트 | ★ |
| SSE 연결 끊김 | 브라우저 자동 재연결 < 5초 | E2E 자동 테스트 | ★ |
| 인메모리 버퍼 오버플로우 | 샘플링 정책 동작 + 에러 로그 기록 | 부하 테스트 초과 주입 | ★ |
| 디스크 쓰기 실패 | 수신 거부 + 알림 발생 (데이터 무손실) | 디스크 full 시뮬레이션 | ★ |

### 4.2 데이터 정합성 (Data Integrity)

| 항목 | 합격 기준 | 우선순위 |
|------|---------|:-------:|
| 수집된 트레이스 완전성 | 손실율 < 0.1% (정상 부하 기준) | ★ |
| 메트릭 타임스탬프 정확성 | ±1초 이내 | ★ |
| 스팬 부모-자식 관계 정확성 | 100% (단일 서버 내) | ★ |
| 24시간 연속 운영 데이터 누락 | 0건 (정상 환경) | ★ |
| v0.9 → v1.0 업그레이드 후 기존 데이터 접근 | 정상 조회 가능 | ★ |
| Prometheus → 자체 스토리지 전환 후 수치 diff | ±5% 이내 (동일 쿼리 기준) | ★ |

### 4.3 운영 안정성

| 항목 | 합격 기준 | 우선순위 |
|------|---------|:-------:|
| 72시간 무중단 운전 | 메모리 누수 없음 (증가량 < 10MB/day), OOM 없음 | ★ |
| Graceful Shutdown | 진행 중 요청 완료 후 종료 (< 30초) | ★ |
| 로그 레벨 런타임 변경 | 재시작 없이 적용 | ☆ |
| `/metrics` 자체 헬스 노출 | Prometheus 포맷 노출 | ☆ |
| Health Check 엔드포인트 | `/health/live`, `/health/ready` 응답 | ★ |
