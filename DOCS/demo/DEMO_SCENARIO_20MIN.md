## 시연 전 준비사항

```bash
# 시연 10분 전 — 모든 서비스 기동 확인
docker compose -f docker-compose.demo.yaml ps    # 모두 Up(healthy)
curl -s http://localhost:8081/api/hello           # Java OK
curl -s http://localhost:8084/api/hello           # Python OK
curl -s http://localhost:11434/api/tags | jq '.models|length'  # Ollama 모델 수

# 부하 발생 (시연 5분 전)
k6 run ~/demo/load/k6-demo.js &
k6 run ~/demo/load/k6-rag.js &

# Celery 배치 워커 실행 확인
curl http://localhost:5555  # Flower UI

# 브라우저 탭 순서 준비
# 탭 1: http://localhost:3000              (AITOP 메인)
# 탭 2: http://localhost:3000/xlog         (XLog)
# 탭 3: http://localhost:3000/heatmap      (히트맵)
# 탭 4: http://localhost:16686             (Jaeger)
# 탭 5: http://localhost:3000/flame        (플레임그래프)
# 탭 6: http://localhost:3000/ai-monitor   (AI 모니터링)

# 탭 8: http://localhost:3000/diagnostic   (진단 모드)
# 탭 9: http://localhost:3000/fleet        (Fleet)
# 탭 10: http://localhost:3000/gpu         (GPU)
```

**[화면: XLog 탭]**

> **말할 내용:**

>

> 빨간 점들은 2초 이상 걸린 요청입니다.
> 이 `/api/slow` 엔드포인트는 의도적으로 느리게 만든 겁니다.
>
> 실무에서는 배포 직후 XLog가 갑자기 빨간 점이 많아지면
> 배포에서 문제가 생긴 거라고 바로 알 수 있습니다."

**[화면 조작]**:
1. 상단의 시간 범위 셀렉터를 '최근 1분'으로 변경
2. 빨간 점 클릭 → 트레이스 팝업 확인
3. 필터 버튼 클릭 → `/api/slow` 패턴으로 필터링

> "서비스별로 필터링도 됩니다. Java만 보거나, 특정 엔드포인트만 볼 수 있습니다."

> **말할 내용:**

## 예상 질문 & 답변 (기술 심화)

### Q1: "Agent가 프로덕션 성능에 영향을 주지 않나요?"

> "실측 기준으로 Java Agent는 CPU +2~3%, 메모리 +80MB 수준입니다.
> 샘플링 레이트를 조정할 수 있어서 트래픽이 많은 환경에서는 1% 샘플링도 가능합니다.
> 진단 모드는 30분 단위로만 켜는 것을 권장합니다.
> 저희 레퍼런스 고객사 중 일 1억 건 트래픽 처리 중에도 안정적으로 운영 중입니다."

### Q2: "기존 Jaeger가 있는데 교체해야 하나요?"

> "교체 필요 없습니다. AITOP의 Collection Server가 기존 Jaeger의 데이터를 읽거나,
> Agent가 두 곳으로 동시에 트레이스를 보낼 수 있습니다.
> 기존 Grafana 대시보드도 그대로 사용하면서 AITOP을 추가하는 방식이 가능합니다."

### Q3: "Kubernetes 환경에서는 어떻게 배포하나요?"

> "Helm Chart가 준비되어 있습니다. `helm install aitop ./helm/` 한 줄이면 됩니다.
> Agent는 DaemonSet으로 배포하거나, 사이드카 패턴도 지원합니다.
> Kubernetes 서비스 메시(Istio, Linkerd) 환경에서도 동작합니다."

### Q4: "멀티 클러스터 환경은 지원하나요?"

> "네, Fleet 관리가 그 용도입니다. 각 클러스터에 Collection Agent를 띄우면
> 중앙 Collection Server로 집계됩니다.
> AWS + On-prem 혼합 환경도 지원합니다."

### Q5: "데이터 보존 기간은 얼마나 되나요?"

> "기본 설정은 7일입니다. Prometheus retention 설정으로 조정 가능합니다.
> 장기 보관이 필요하면 Thanos나 Cortex와 연동할 수 있습니다.
> 트레이스는 스토리지 비용이 크므로 보통 3~7일, 메트릭은 30~90일로 설정합니다."

### Q6: "LLM API (OpenAI, Claude) 호출도 모니터링되나요?"

> "네, HTTP 레이어에서 자동으로 잡힙니다.
> OpenAI API 응답 헤더에 있는 토큰 사용량도 파싱해서 보여줍니다.
> 모델별, 사용자별 토큰 비용 집계도 됩니다."

### Q7: "알림을 PagerDuty 말고 다른 곳으로 보낼 수 있나요?"

> "Webhook 기반이라 Slack, Teams, PagerDuty, OpsGenie, 이메일, SMS 모두 지원합니다.
> 커스텀 Webhook으로 사내 시스템 연동도 가능합니다."

### Q8: "플레임그래프 수집이 상시 켜져 있으면 오버헤드가 크지 않나요?"

> "Async Profiler 기반으로 100ms 간격 샘플링을 씁니다.
> CPU 오버헤드는 1% 미만입니다. 메모리 프로파일링은 기본 꺼져 있고
> 필요할 때만 켭니다. 진단 모드에서 더 세밀한 수집이 가능합니다."

```bash

k6 run -u 100 -d 30s ~/demo/load/k6-error.js

# 2. AITOP 알림 발생 확인 (에러율 급등)
# 3. XLog에서 에러 요청 클릭
# 4. 트레이스에서 RuntimeException 스택 트레이스 확인
# 5. 코드 수정 (또는 롤백) → 에러 감소 확인
```
