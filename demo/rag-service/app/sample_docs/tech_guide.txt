AI/ML 기술 가이드 — 개발자를 위한 핵심 개념 정리

1. 대규모 언어 모델 (LLM) 이해하기

LLM(Large Language Model)은 대량의 텍스트 데이터를 학습하여
자연어를 이해하고 생성할 수 있는 딥러닝 모델입니다.

1-1. 주요 아키텍처
- Transformer: 어텐션 메커니즘을 사용하여 문맥을 파악
- GPT 계열: 디코더 기반, 텍스트 생성에 최적화
- BERT 계열: 인코더 기반, 텍스트 이해/분류에 최적화
- T5/BART: 인코더-디코더 기반, 번역/요약에 적합

1-2. 주요 성능 지표
- TTFT (Time To First Token): 첫 번째 토큰이 생성되기까지의 시간
  사용자 체감 응답 속도를 결정하는 핵심 지표입니다.
  목표: P95 기준 2초 이내
- TPS (Tokens Per Second): 초당 생성되는 토큰 수
  스트리밍 응답의 유창함을 결정합니다.
  목표: P50 기준 30 tok/s 이상
- Throughput: 단위 시간당 처리할 수 있는 총 요청 수

2. RAG (Retrieval-Augmented Generation)

RAG는 LLM의 지식 한계를 보완하기 위해 외부 문서를 검색하여
프롬프트에 포함시키는 기법입니다.

2-1. RAG 파이프라인 구조
1) 문서 전처리: 원본 문서를 청크(chunk)로 분할
2) 임베딩: 각 청크를 벡터(숫자 배열)로 변환
3) 인덱싱: 벡터를 벡터 DB에 저장
4) 검색: 사용자 질문을 임베딩하여 유사 벡터 검색
5) 생성: 검색된 문서와 질문을 LLM에 전달하여 답변 생성

2-2. 임베딩 모델 선택
- OpenAI text-embedding-3-small: 범용, API 기반
- sentence-transformers: 오픈소스, 로컬 실행 가능
- BGE/E5: 다국어 지원, 한국어 성능 우수

2-3. 벡터 데이터베이스 비교
- Pinecone: 완전 관리형 SaaS, 빠른 시작
- Qdrant: 오픈소스, 필터링 강력, 자체 호스팅
- Milvus: 대규모 벡터 처리, GPU 가속 지원
- Chroma: 경량, 프로토타이핑에 적합

3. 프롬프트 엔지니어링

3-1. 기본 원칙
- 명확한 지시: "요약해줘" 보다 "3문장으로 요약해줘"가 효과적
- 역할 부여: "당신은 경험 많은 프로젝트 매니저입니다"
- 예시 제공: Few-shot learning으로 출력 형식 안내
- 제약 조건: "200자 이내", "JSON 형식으로"

3-2. RAG 프롬프트 템플릿
제공된 문서 정보만을 기반으로 답변하세요.
문서에 없는 내용은 "해당 정보를 찾을 수 없습니다"라고 답하세요.

4. AI 서비스 모니터링 핵심

4-1. 왜 AI 서비스 모니터링이 중요한가
- 전통적 웹 서비스와 달리 AI 서비스는 비결정적(non-deterministic)
- GPU 리소스 비용이 높아 효율적 사용이 중요
- TTFT가 사용자 이탈률에 직접적 영향
- 가드레일 실패 시 법적/윤리적 리스크

4-2. 핵심 모니터링 포인트
- 추론 서버 큐 대기 시간
- GPU VRAM 사용률 (90% 초과 시 OOM 위험)
- 토큰 생성 속도 (TPS) 변동 추이
- 가드레일 차단율 및 응답 시간
- 벡터 검색 레이턴시 및 검색 품질
