# UI 화면 테스트 체크리스트

> **프로젝트**: AITOP AI Service Monitoring Platform
> **목적**: 43개 전체 페이지를 데모 데이터 기반으로 화면 렌더링·기능·UX를 점검
> **실행 환경**: Frontend 개발 서버 (`npm run dev`) + 데모 데이터 (`demo-data.ts`)
> **브라우저**: Chrome 최신 (권장), F12 개발자 도구 콘솔 열어둘 것

---

## 시작 전 준비

### 1. Frontend 개발 서버 기동

```bash
# Bash (Git Bash / WSL2)
cd /c/workspace/aiservice-monitoring/frontend && npm run dev
```

```powershell
# PowerShell
cd C:\workspace\aiservice-monitoring\frontend; npm run dev
```

- `http://localhost:3000` 접속 확인
- 포트 충돌 시: `npm run dev -- -p 3001`

### 2. 데모 로그인

| 역할 | 이메일 | 비밀번호 | 용도 |
|------|--------|---------|------|
| **Administrator** | `admin@aitop.io` | `admin` | 전체 기능 접근 |
| **SRE** | `sre@aitop.io` | `sre` | 인시던트 대응 플로우 |
| **AI Engineer** | `ai@aitop.io` | `ai` | AI 서비스 튜닝 플로우 |
| **Viewer** | `viewer@aitop.io` | `viewer` | 읽기 전용 접근 |

### 3. 점검 방법

각 페이지에서 아래 5가지를 확인합니다:

| 기호 | 점검 항목 | 설명 |
|:----:|----------|------|
| **R** | 렌더링 | 페이지가 빈 화면/에러 없이 표시되는가 |
| **D** | 데이터 | 데모 데이터가 차트/테이블/카드에 정상 표시되는가 |
| **I** | 인터랙션 | 클릭/호버/필터/정렬이 동작하는가 |
| **C** | 콘솔 | F12 콘솔에 빨간 에러가 없는가 |
| **M** | 모바일 | 반응형 레이아웃이 깨지지 않는가 (F12 → 모바일 뷰) |
